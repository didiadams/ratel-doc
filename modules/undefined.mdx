---
title: 'Tenant Management'
---

**Purpose**: Manage bank/institution tenants on the platform (multi-tenant SaaS).

| Action | Endpoint | Who Can Do It |
|--------|----------|---------------|
| Create tenant | `POST /api/v1/tenants` | SUPER_ADMIN only |
| List tenants | `GET /api/v1/tenants` | SUPER_ADMIN only |
| Get tenant | `GET /api/v1/tenants/:id` | SUPER_ADMIN, BANK_ADMIN |
| Update tenant | `PATCH /api/v1/tenants/:id` | SUPER_ADMIN only |

**Create Tenant Request:**
```json
{
  "name": "First Bank of Nigeria",
  "slug": "first-bank",
  "adminEmail": "compliance@firstbank.com",
  "adminFirstName": "Adaeze",
  "adminLastName": "Okonkwo",
  "adminPassword": "SecurePass123!"
}
```

Each tenant gets:
- Isolated data (transactions, cases, rules, reports)
- Their own admin user
- Default CBN rules (auto-seeded)
- Independent engine configuration
